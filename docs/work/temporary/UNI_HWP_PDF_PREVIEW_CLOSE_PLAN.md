# Uni-HWP PDF Preview Close Plan

## 1. 작업 목표

PDF 미리보기 창의 우측 상단 `[x]` 버튼이 앱 종료가 아닌 `미리보기 닫기 -> 편집기 복귀` 역할을 수행하도록 수정하되, 기존 편집기 복귀 흐름과 정리 로직을 재사용해 UX를 일관되게 만든다.

## 2. 작업 원칙

- RHWP 엔진 코어는 건드리지 않는다.
- 기존 PDF 미리보기 컨트롤러와 편집기 복귀 로직을 최대한 재사용한다.
- 창 닫기 동작은 새로운 분기 로직을 늘리기보다 기존 복귀 동작에 합류시키는 방향으로 구현한다.

## 3. 구현 범위

- PDF 미리보기 관련 컨트롤러/이벤트 처리 코드
- Tauri 또는 브라우저 창 닫기 요청 처리 코드
- 편집기 복귀 및 미리보기 정리 코드

## 4. 작업 단계

### 4.1 현행 구조 확인

- PDF 미리보기 창이 별도 창인지, 동일 창 내 전환인지 확인
- 현재 `[x]`가 무시되는 이유가 `closeRequested preventDefault`, 라우팅 가드, 미리보기 보호 로직 중 어디인지 파악
- 기존 `뒤로가기` 또는 `편집기로 복귀`가 호출하는 정리 함수 확인

### 4.2 공통 닫기 경로 정리

- `[x]` 이벤트가 기존 복귀 함수와 동일한 공통 종료 함수로 들어가도록 정리
- 닫기 시 필요한 임시 상태 해제, 리스너 제거, URL 정리, 미리보기 DOM 제거를 한 곳에서 처리
- 좌측 `뒤로가기` 버튼은 유지하고, 동일 공통 종료 함수로 연결

### 4.3 UX 보정

- `[x]` 클릭 시 무반응처럼 보이지 않도록 즉시 처리
- 필요 시 닫기 불가 가드를 제거하거나, PDF 미리보기 전용으로 우회
- 좌측 `뒤로가기` 버튼은 내비게이션 의미를 유지하되, 우측 상단 `[x]`와 결과적으로 같은 복귀 동작을 수행하도록 정리

### 4.4 검증

- PDF 미리보기 `[x]` 클릭 시 정상 복귀 확인
- 내부 복귀 버튼 동작 유지 확인
- 반복 실행 시 상태 누수 없는지 확인

## 5. 검증 항목

- `인쇄 -> PDF 내보내기 -> 인쇄` 후 열린 미리보기에서 `[x]` 클릭
- 편집기로 정상 복귀 여부
- 복귀 후 다시 동일 경로 실행 가능 여부
- 복귀 버튼과 `[x]` 버튼의 동작 결과 동일 여부

## 6. 완료 조건

- `[x]`가 실질적인 닫기/복귀 동작을 수행
- `[x]`는 앱 종료가 아니라 PDF 미리보기 종료로 동작
- 좌측 `뒤로가기` 버튼은 유지되며 기존 복귀 의미를 계속 제공
- 기존 복귀 UX와 충돌 없음
- 재진입 시 오류 및 잔여 상태 없음
