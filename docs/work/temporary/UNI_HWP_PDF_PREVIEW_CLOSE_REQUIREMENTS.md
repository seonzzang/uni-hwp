# Uni-HWP PDF Preview Close Requirements

## 1. 목적

`인쇄 -> PDF 내보내기 -> 인쇄` 경로에서 열리는 PDF 미리보기 창의 우측 상단 `[x]` 버튼이 사용자의 기대대로 동작하도록 UX를 정정한다.

현재는 `[x]` 버튼이 닫기 역할을 수행하지 않아, 사용자가 앱이 멈췄거나 닫기 불가 상태라고 오해할 수 있다.

## 2. 대전제

- RHWP 엔진 코어는 수정하지 않는다.
- 수정 범위는 Uni-HWP 앱 셸, PDF 미리보기 컨트롤러, 창 닫기 이벤트 처리 계층으로 한정한다.
- 기존 `뒤로가기` 또는 `편집기로 복귀` 흐름과 충돌하지 않아야 한다.

## 3. 기능 요구사항

### 3.1 닫기 버튼 동작

- PDF 미리보기 창의 우측 상단 `[x]` 버튼은 정상적으로 닫기 동작을 수행해야 한다.
- PDF 미리보기 화면의 우측 상단 `[x]` 버튼은 앱 종료가 아니라 PDF 미리보기 세션 종료를 의미한다.
- 사용자가 `[x]`를 누르면 앱 전체가 종료되지 않고 편집기로 복귀해야 한다.
- 닫기 결과는 사용자가 `뒤로가기` 또는 `편집기로 복귀`를 수행한 것과 기능적으로 동일해야 한다.

### 3.2 복귀 흐름 일관성

- `[x]` 버튼으로 닫을 경우에도 편집기 화면으로 안전하게 복귀해야 한다.
- PDF 미리보기용 임시 상태, 오버레이, 이벤트 리스너, 임시 URL 또는 임시 파일 정리 흐름이 누락되면 안 된다.
- 닫기 후 다시 인쇄 또는 PDF 내보내기를 실행해도 이전 미리보기 상태가 남아 있지 않아야 한다.
- 좌측 `뒤로가기` 버튼은 유지한다.
- 좌측 `뒤로가기` 버튼과 우측 상단 `[x]` 버튼은 표현은 다르지만 결과적으로 동일한 복귀 흐름을 수행해야 한다.
- 좌측 버튼은 내비게이션 의미, 우측 상단 `[x]`는 닫기 의미를 전달해야 한다.

### 3.3 미저장 문서 보호와의 분리

- PDF 미리보기 창 닫기는 편집 문서의 종료와 동일한 dirty 저장 확인 대상으로 취급하지 않는다.
- PDF 미리보기는 보조 뷰이므로, 닫기 자체에 추가 저장 확인 팝업을 띄우지 않는다.

## 4. 비기능 요구사항

- 사용자는 `[x]` 버튼을 눌렀을 때 즉시 반응한다고 느껴야 한다.
- 닫기 이벤트 처리로 인해 프리징처럼 보이는 지연이 생기면 안 된다.
- Windows 타이틀바 닫기 버튼, 내부 복귀 버튼, 키보드 기반 닫기 흐름 간 일관성을 유지해야 한다.

## 5. 제외 범위

- RHWP 렌더링 엔진 수정
- PDF 생성 파이프라인 구조 개편
- PDF 뷰어 전체 UI 재설계

## 6. 완료 판정

- PDF 미리보기 창에서 `[x]` 클릭 시 창이 정상 종료되거나 편집기로 정상 복귀한다.
- `뒤로가기` 및 기존 복귀 동작과 결과가 충돌하지 않는다.
- 닫은 뒤 동일 문서에서 다시 PDF 미리보기를 실행해도 정상 동작한다.
